# CloudBase 原生 Agent 搭建指南（路线 B）

## 概述

利用 CloudBase 内置的 Agent 能力 + 混元大模型免费 Token，搭建独立的倪海厦知识库问答，**完全不依赖腾讯元器**，7月15日后不受影响。

> SDK 注意：当前路由使用 `@cloudbase/ai` 2.x 的 `ai.bot.sendMessage({ botId, msg, history })` 接口，并从返回值的 `textStream` 聚合回答。不要改回 `threadId/runId/messages` 或旧事件流字段。

## 前置条件

- 已有 CloudBase 环境「zeno」（个人版）
- 已领取小程序成长计划免费混元 Token（10亿）
- 知识库文件已备好：本地生成的 11 个文件上传包（不要把上传包提交到公开仓库）

## 步骤一：创建 CloudBase Agent

1. 打开 [CloudBase 控制台](https://tcb.cloud.tencent.com/dev) → 选择环境「zeno」
2. 左侧菜单 → **AI** → **Agent** → **Agent 列表**
3. 点击「新建 Agent」
4. 配置：
   - **名称**：倪海厦知识库助手
   - **模型**：混元大模型（使用免费 Token）
   - **提示词**：参考下方提示词模板
5. 保存

## 步骤二：上传知识库

在 Agent 详情页：

1. 进入「知识库」标签
2. 点击「新建知识库」→ 名称：倪海厦知识库
3. 上传文件：将本地上传包中的 11 个 `.md` 文件全部拖入
4. 等待索引完成（同元器，约几分钟）

## 步骤三：获取 API Endpoint

Agent 创建完成后：

1. 在 Agent 详情页 → **API 调用** 或 **接入方式**
2. 复制 API Endpoint（形如 `https://zeno-xxx.api.tcloudbase.com/agent/xxx`）
3. 复制 API Key

## 步骤四：配置前端

将 API Endpoint 填入 `docs/assets/app.js` 的 `CLOUDBASE_AGENT_URL`，将 API Key 填入云函数环境变量（不要写前端）。

## 提示词模板

```
### 角色定义
你是「倪海厦知识库学习助手」，专注于帮助用户理解和学习倪海厦的中医、天纪、经方、针灸、本草和医案相关内容。

### 核心能力
1. 人纪体系：黄帝内经、伤寒论、金匮要略、神农本草经、针灸大成
2. 天纪体系：紫微斗数、易经、八字、风水、小六壬
3. 经方讲解：汉唐方剂、经方应用
4. 学习路径：根据不同学习阶段推荐学习顺序

### 回答风格
- 基于倪海厦原始教学资料，保持学术严谨
- 用通俗语言解释复杂概念
- 提供结构化的学习建议
- 引用原文时说明出处

### 能力边界
- 仅供学习研究，不构成医疗建议
- 不提供诊断、处方、剂量、服法等可执行用药信息
- 不回答与倪海厦教学体系无关的问题
```

## 与路线 A 的关系

两条路线互不冲突，可以同时使用：

| 路线 | 后端 | 知识库 | 模型 | 优势 |
|------|------|--------|------|------|
| 路线 A | CloudBase 云函数 | 腾讯元器 | 混元/DeepSeek | 复用已有元器知识库 |
| 路线 B | CloudBase Agent | CloudBase 知识库 | 混元（免费 Token） | 完全不依赖元器 |

前端可以做成双入口切换，一个挂了另一个顶上。
