# 倪海厦知识图谱 - 国内镜像部署说明

## 1. 推荐部署方式

**推荐：阿里云 OSS 静态网站托管**

原因：
- 本站是纯静态网站（HTML + CSS + JS + JSON）
- 不需要服务器，无需运维
- 国内访问稳定，速度快
- 后续可无缝接入 CDN 加速和自定义域名

## 3. OSS 设置步骤

### 3.1 创建 Bucket

1. 登录 [阿里云 OSS 控制台](https://oss.console.aliyun.com/)
2. 点击「创建 Bucket」
3. Bucket 名称：自定义，例如 `nihaixia-knowledge-graph`
4. 地域选择：
   - 华东地区（杭州/上海）：适合江浙沪用户
   - 华南地区（深圳/广州）：适合南方用户
   - 华北地区（北京）：适合北方用户
   - 按目标读者主要分布区域选择
5. 存储类型：标准存储
6. 读写权限：按阿里云静态网站托管要求配置最小可公开读取权限

### 3.2 开启静态网站托管

1. 进入 Bucket 详情页
2. 左侧菜单选择「基础设置」>「静态网站」
3. 点击「开启」
4. 设置：
   - 默认首页：`index.html`
   - 默认 404 页面：留空（本项目使用前端路由）
   - 子目录首页：留空
5. 保存设置

### 3.3 上传文件

**方法一：控制台上传**
1. 进入 Bucket「文件管理」
2. 直接拖拽解压后的文件夹内容上传
3. 保持目录结构不变

**方法二：使用 ossutil 命令行工具**
```bash
# 安装 ossutil
# macOS
brew install ossutil

# 或下载安装
# https://help.aliyun.com/zh/oss/ossutil-install/

# 配置
ossutil config

# 上传整个目录
ossutil cp -r ./nihaixia-knowledge-graph-docs-cn-mirror/ oss://your-bucket-name/ --update
```

### 3.4 设置文件权限

静态网站托管的文件需要可公开访问。建议按以下方式配置：

1. 优先按阿里云控制台提示配置最小可公开读取权限（如 Bucket Policy 或静态网站托管的默认权限）
2. 如果控制台要求公共读或 Bucket Policy，只开放网站静态文件访问即可
3. **不要上传 `sources-review/`、私密资料、原始高风险材料到此 Bucket**

更详细的权限配置请参考 [阿里云 OSS 静态网站托管文档](https://help.aliyun.com/zh/oss/user-guide/host-a-static-website.html)。

## 5. 域名和备案说明

### 5.1 使用 OSS 默认地址

- OSS 静态网站提供默认域名：`your-bucket-name.oss-cn-xxx.aliyuncs.com`
- 可用于测试或临时镜像
- 正式公开访问、自定义域名、CDN 加速、长期运营，请以阿里云控制台和备案规则为准
- 中国内地自定义域名 / CDN 通常需要 ICP 备案

### 5.2 绑定自定义域名（需要备案）

如果要绑定自己的域名（如 `nihaixia.example.com`）：

1. **ICP 备案要求：**
   - Bucket 在中国内地 → 绑定的域名必须完成 ICP 备案
   - Bucket 在中国香港或其他地区 → 不需要备案（但国内访问速度可能较慢）

2. **备案流程：**
   - 通过阿里云或其他备案服务商提交备案
   - 个人网站备案通常需要 1-3 周
   - 备案成功后，在 OSS 控制台绑定自定义域名

3. **绑定域名步骤：**
   - 在 OSS 控制台「基础设置」>「域名管理」添加自定义域名
   - 配置 CNAME 解析指向 OSS 域名
   - 等待 DNS 生效

### 5.3 接入 CDN（推荐，需要备案）

国内 CDN 加速域名通常也需要 ICP 备案：

1. 在阿里云 CDN 控制台创建加速域名
2. 源站类型选择「OSS Bucket」
3. 填写 Bucket 域名
4. CDN 域名需要完成备案
5. 配置缓存策略（静态资源建议缓存 7-30 天）
6. 开启 HTTPS（推荐使用免费证书）

## 7. 常见问题

### Q1: 为什么选择 OSS 而不是直接用 GitHub Pages？

GitHub Pages 在国内访问不稳定，有时会完全无法访问。OSS 是阿里云的静态存储服务，国内访问稳定快速。

### Q2: 需要服务器吗？

不需要。OSS 静态网站托管是纯静态服务，所有逻辑都在前端 JavaScript 中完成。

### Q3: 更新内容怎么办？

1. 修改 `docs/` 目录中的文件
2. 重新运行打包命令
3. 上传新的 zip 包到 OSS

### Q4: 如何查看访问日志？

在 OSS 控制台「日志管理」中开启访问日志，日志会存储到指定的 Bucket 中。

### Q5: 费用如何？

- OSS 存储费用：非常低，70 个节点的网站数据量很小
- 流量费用：按实际访问量计费，初期几乎可忽略
- CDN 费用（如果使用）：有免费额度，超出后按流量计费
